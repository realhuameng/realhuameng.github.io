    Git 是一个开源的分布式的版本控制系统

    工作流程

    克隆Git资源位工作目录

    在克隆的资源上添加或修改文件

    更新资源

    在提交前查看修改状态

    提交修改

    修改完成后，可以撤回提交并再次修改提交

    Git工作区、暂存区、版本库

    工作区：电脑里能看到的目录

    暂存区：stage 或 index，一般放在 .git 目录下的index文件（**.git/index**）中，也叫做索引

    版本库：工作区有一个隐藏目录，**.git** ，实际上是Git的本地版本仓库

    Git基本命令

    创建版本库

    创建一个gitspace空目录（可选），用作以后创建本地版本仓库

    $ mkdir test #创建目录
$ cd test #进入目录
$ pwd
/d/gitspace/test
$ git init #初始化当前仓库
Initialized empty Git respository in D:/gitspace/test/.git

    .git 目录是Git用来跟踪管理版本仓库的

    可使用ls -ah查看隐藏目录

    新建文件

    -add：将文件添加到缓存区

    -commit：提交到本地仓库

    $ touch test.c #创建新文件
$ git add test.c #将文件添加到本地仓库的提交缓存
#简单描述我们做了什么 使用-m命令简写描述信息
$ git commit -m "add new file \"test.c\""
$ git status

    版本回退

    git log命令显示从最近到最远的提交日志

    $ git log #查看日志记录
$ git log --pretty=oneline
    $ git reset --hard HEAD^ #回到第一个版本
$ git reset --hard 7fs9vj47 #版本号写出前8位即可

    Git内部有个指向当前版本的HEAD指针，当回退版本时，Git仅仅是把HEAD指向当前的版本

    $ git log #查看日志
$ git reflog #展示每次的命令，commit id

    Git的管理和修改

    $ git diff HEAD -- 文件名 #对比工作区和本地仓库中文件的区别
$ git restore 文件名 #撤销文件在工作区做的修改
$ git checkout -- 文件名 #不能撤销已经添加到暂存区的内容
    $ git restore --staged 文件名 #从暂存区撤回，退回到工作状态
$ git reset HEAD 文件名

    Git删除文件

    $ rm 文件名 #仅删除工作区的文件
    $ git rm 文件名 #同时删除工作区和暂存区的文件
$ git commit -m "commit del 文件名" #将删除修改提交到版本库中
